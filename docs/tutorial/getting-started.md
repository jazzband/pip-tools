# Getting started

This tutorial takes you from zero to a pinned, syncable project in about ten minutes. You will use both
commands `pip-tools` ships today: `pip-compile` to write a lockfile, and `pip-sync` to make a virtual
environment match it.

## Prerequisites

- Python 3.10 or later.
- `pip-tools` installed. {doc}`/how-to/install` covers `uv`, `pipx`, and `pip` setups.

This tutorial uses a fresh project directory. Create one:

```console
mkdir myapp && cd myapp
```

## The loop you will learn

```{mermaid}
flowchart LR
    A[Edit requirements.in] --> B[pip-compile]
    B --> C[requirements.txt]
    C --> D[pip-sync]
    D --> E[venv ready]
    E -.->|need a new package| A

    style A fill:#2563eb,stroke:#1d4ed8,color:#fff
    style B fill:#6366f1,stroke:#4f46e5,color:#fff
    style C fill:#16a34a,stroke:#15803d,color:#fff
    style D fill:#6366f1,stroke:#4f46e5,color:#fff
    style E fill:#16a34a,stroke:#15803d,color:#fff
```

You write top-level dependencies in `requirements.in`. `pip-compile` resolves them and writes
`requirements.txt`. `pip-sync` makes the venv match. To add a new package, edit `requirements.in` and run
the loop again.

## Step 1: declare your dependencies

Create a `requirements.in` file:

```text
flask
requests
```

Two top-level packages, no versions. `pip-compile` will pick versions for you.

## Step 2: create a virtual environment

::::{tab} Linux/macOS

```console
python -m venv .venv
source .venv/bin/activate
```

::::

::::{tab} Windows (PowerShell)

```console
python -m venv .venv
.venv\Scripts\Activate.ps1
```

::::

::::{tab} Windows (cmd)

```console
python -m venv .venv
.venv\Scripts\activate.bat
```

::::

## Step 3: compile

From the project root:

```console
pip-compile
```

`pip-compile` discovers `requirements.in`, resolves it, and writes `requirements.txt`:

```text
#
# This file is autogenerated by pip-compile with Python 3.13
# by the following command:
#
#    pip-compile
#
blinker==1.7.0
    # via flask
certifi==2024.2.2
    # via requests
charset-normalizer==3.3.2
    # via requests
click==8.1.7
    # via flask
flask==3.0.2
    # via -r requirements.in
idna==3.6
    # via requests
itsdangerous==2.1.2
    # via flask
jinja2==3.1.3
    # via flask
markupsafe==2.1.5
    # via
    #   jinja2
    #   werkzeug
requests==2.31.0
    # via -r requirements.in
urllib3==2.2.1
    # via requests
werkzeug==3.0.1
    # via flask
```

The two top-level packages produced eleven pinned lines. Every transitive dependency gets a `# via`
annotation pointing at where it came from.

## Step 4: sync the environment

```console
pip-sync
```

`pip-sync` reads `requirements.txt`, compares it to the empty venv, and installs everything:

```text
Collecting blinker==1.7.0
  ...
Successfully installed blinker-1.7.0 certifi-2024.2.2 ... werkzeug-3.0.1
```

Verify it worked:

```console
(.venv) $ python -c "import flask, requests; print(flask.__version__, requests.__version__)"
3.0.2 2.31.0
```

## Step 5: add a dependency

Edit `requirements.in` and add a third package:

```text
flask
requests
gunicorn
```

Compile again:

```console
pip-compile
```

The diff in `requirements.txt`:

```diff
+gunicorn==21.2.0
+    # via -r requirements.in
+packaging==24.0
+    # via gunicorn
```

Two new lines: `gunicorn` itself, and its one transitive dependency. Every other package kept its pinned
version. `pip-compile` does not upgrade anything you did not ask to upgrade. See
{doc}`/explanation/stable-output` for why.

Sync the environment to install the new package:

```console
(.venv) $ pip-sync
Collecting gunicorn==21.2.0
  ...
Collecting packaging==24.0
  ...
Successfully installed gunicorn-21.2.0 packaging-24.0
```

## Step 6: bump a dependency

Time passes. A new `requests` ships and you want it. Bump just that package:

```console
pip-compile -P requests
```

`-P` (`--upgrade-package`) re-resolves only `requests` and leaves every other pin alone. The diff:

```diff
-requests==2.31.0
+requests==2.32.0
```

Sync to install the new version:

```console
(.venv) $ pip-sync
Collecting requests==2.32.0
  ...
Successfully installed requests-2.32.0
```

## Step 7: remove a dependency

Decide you don't need `requests` after all. Edit `requirements.in`:

```text
flask
gunicorn
```

Compile:

```console
pip-compile
```

The diff:

```diff
-charset-normalizer==3.3.2
-    # via requests
-idna==3.6
-    # via requests
-requests==2.31.0
-    # via -r requirements.in
-urllib3==2.2.1
-    # via requests
```

`requests` and its transitive dependencies are gone from `requirements.txt`.

Sync:

```console
(.venv) $ pip-sync
Found existing installation: requests 2.32.0
Uninstalling requests-2.32.0:
  Successfully uninstalled requests-2.32.0
... (also uninstalls charset-normalizer, idna, urllib3)
```

`pip-sync` uninstalled the four packages that are no longer in the lockfile.

## What you learned

- `requirements.in` declares what you want.
- `pip-compile` resolves it into `requirements.txt`.
- `pip-sync` makes the venv match.
- Re-compiling without `-P` or `-U` does not move pins you did not ask to move.
- `pip-sync` uninstalls packages no longer in the lockfile.
